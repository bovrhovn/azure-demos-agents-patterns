## Guidelines

- Write documentation in Markdown.
- Keep documentation close to the code it describes — prefer `README.md` files inside [`src/`](../src/README.md) sub-folders for pattern-specific docs.
- Use this directory for cross-cutting concerns: architecture, ADRs (Architecture Decision Records), contribution guides, and deployment runbooks.
